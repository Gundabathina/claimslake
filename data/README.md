# data/

This folder holds small **synthetic** sample data files used for local development, testing, and demos.

- All data here is generated by `scripts/generate_synthetic_data.py` (see Milestone 1). It does not contain any real patient, member, or provider information.
- Large generated datasets are not committed to Git; only small representative samples are kept here so the repository stays lightweight and clonable.
- Full-size generated data is written locally to `data/generated/` (gitignored) or to the MinIO `bronze` bucket when running the pipeline end-to-end.

## Structure (populated in later milestones)

- `sample/` — small CSV/Parquet samples of each source entity (members, providers, diagnoses, claims) for quick inspection and unit tests.
